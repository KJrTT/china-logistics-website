Корпоративный B2B сайт-визитка для компании, оказывающей услуги по организации поставок товаров из Китая.

Проект разработан в рамках производственной практики и предназначен для представления компании в сети Интернет, информирования потенциальных клиентов об оказываемых услугах и автоматизации процесса приема заявок через форму обратной связи.

# Описание проекта

Проект представляет собой корпоративный многостраничный сайт-визитку логистической компании, специализирующейся на организации поставок товаров из Китая.

Сайт позволяет потенциальным клиентам ознакомиться с информацией о компании, перечнем услуг, преимуществами сотрудничества и отправить заявку на консультацию через форму обратной связи.

Для обработки заявок реализована серверная часть на FastAPI, обеспечивающая прием данных, их проверку, сохранение в базе данных и отправку email-уведомлений.

---

# Предметная область

Компания осуществляет поиск поставщиков, организацию международной логистики, сопровождение внешнеэкономической деятельности и помощь в поставках товаров из Китая.

Одной из задач бизнеса является привлечение новых клиентов через Интернет и обеспечение удобного способа подачи заявок на сотрудничество.

Для решения данной задачи был разработан корпоративный B2B сайт-визитка с возможностью автоматизированного приема обращений.

# Архитектура системы

```
Пользователь
      │
      ▼
Frontend (HTML/CSS/JS)
      │ AJAX
      ▼
Backend (FastAPI)
   ├── PostgreSQL
   └── SMTP Email
```

Система построена по клиент-серверной архитектуре.

Клиентская часть отвечает за отображение интерфейса и взаимодействие с пользователем.

Серверная часть обеспечивает обработку запросов, сохранение данных и отправку уведомлений.

# Алгоритм обработки заявки

1. Пользователь заполняет форму обратной связи.
2. Выполняется клиентская проверка данных.
3. JavaScript отправляет AJAX-запрос на сервер.
4. FastAPI принимает заявку.
5. Выполняется серверная валидация данных.
6. Заявка сохраняется в базе данных.
7. Отправляется уведомление менеджеру.
8. Пользователю отправляется письмо-подтверждение.
9. Сервер возвращает успешный ответ.
10. На сайте отображается сообщение об успешной отправке.

# База данных

Для хранения заявок используется таблица feedbacks.

| Поле         | Тип      | Назначение           |
| ------------ | -------- | -------------------- |
| id           | Integer  | Идентификатор заявки |
| name         | String   | Имя пользователя     |
| contact      | String   | Email или телефон    |
| message      | Text     | Текст заявки         |
| status       | String   | Статус обработки     |
| is_processed | Boolean  | Признак обработки    |
| created_at   | DateTime | Дата создания        |
| updated_at   | DateTime | Дата изменения       |

# API

## Отправка заявки

POST

```
/api/feedback/
```

Пример запроса:

```
{
  "name": "Иван Иванов",
  "contact": "example@mail.ru",
  "message": "Интересует организация поставок товаров из Китая."
}
```

Пример успешного ответа:

```
{
  "message": "Заявка успешно отправлена"
}
```

# Валидация данных

На стороне клиента и сервера реализованы проверки:

* минимальная длина имени;
* корректность контактных данных;
* минимальная длина сообщения;
* обработка пустых значений;
* контроль формата email;
* контроль номера телефона.

При возникновении ошибок сервер возвращает HTTP 422.
